# 输入法词条评估技能

## 概述

评估文本文件中的词条是否适合列入输入法词典。**必须由LLM对每个词条逐一进行多维度的适用性评估。严禁编写脚本或程序直接输出评分结果。**

**适用范围：** 以中国大陆地区简体中文字形为主，评估词条是否符合简体中文规范和使用习惯。

## 使用场景

- 批量评估词条是否适合输入法词典
- 筛选高质量的输入法候选词
- 检查词条的常用度、规范性和适用性
- 处理大量词条数据时需要LLM辅助判断

## 工作流程

### 1. 确认处理参数

确认以下信息：
- **输入文件**：要处理的文本文件路径
- **输出文件**：评估结果保存的CSV文件路径（如未指定则输出到对话中）
- **批次大小**：每批处理的词条数量（建议20-100条）
- **起始位置**：从第几行开始处理（默认第1行）
- **特殊要求**：是否有特定评估标准

若指定的输出文件已包含内容，需与用户确认：接续之前的进度、覆盖之前的进度，或从头开始但不覆盖之前的内容。

### 2. 读取并评估词条

使用 `read_file` 工具读取指定范围的词条内容。

**对每个词条逐一**按以下维度进行评估：
- **常用性**（高权重）：日常使用频率
- **规范性**（高权重）：是否为标准书写形式，无明显错别字
- **完整性**（中权重）：是否为完整词汇，非残缺片段
- **歧义性**（中权重）：是否容易与其他词混淆
- **专业性**（低权重）：是否为过于专业的术语

详细的评估维度细则、词条分类体系、评分标准、排除标准和特殊词条处理，参见 `references/evaluation-criteria.md`。

### 3. 词条分类

对每个词条进行分类（详见 `references/evaluation-criteria.md`）：
- **基础分类**（必填，单选）：名词/动词/形容词/副词/代词/数词/量词/拟声词/叹词/成语/专有名词/短语/网络用语/其他
- **其他标签**（可选，多选）：#口语 #书面语 #方言 #专业术语 #新词 #生僻 #多音字 #易错 #繁体 #港台用语 #英语 #法语 #德语 #俄语 #韩语 #日语罗马音 #和字

### 4. 输出评估结果

评估结果以CSV格式输出：

```csv
置信度,评分,词条,基础分类,其他标签,原因
9,95,手机,名词,#口语,极常用词汇，简体中文规范
8,88,运行,动词,,常用动词，使用频率高
9,85,June,名词,#英语,常用英语词汇，六月
9,90,Amazon,专有名词,#英语,知名公司名，亚马逊
7,55,畫面,名词,#繁体,繁体字词条，建议转换为"画面"
```

**CSV字段说明：**
- **置信度**：评估置信度（0-10分）
- **评分**：适用性评分（0-100分）
- **词条**：原始词条内容
- **基础分类**：必填分类
- **其他标签**：可选标签，多个标签用空格分隔
- **原因**：简要说明（一句话）

**评分参考：**
- 适用性评分：90-100 强烈推荐 / 70-89 推荐 / 50-69 可考虑 / 30-49 不建议 / 0-29 不推荐
- 置信度评分：8-10 依据充分 / 5-7 较充分 / 0-4 需人工复核

### 5. 汇总统计

每批次结束后输出统计：
- 高分词条（≥70分）数量
- 中分词条（50-69分）数量
- 低分词条（<50分）数量
- 平均评分
- 平均置信度
- 建议操作

### 6. 输出目标

- **指定输出文件**：将CSV结果追加写入指定文件
- **未指定输出文件**：在对话中直接展示CSV内容，用户可复制保存

## 内容中立原则

输入法词典服务于用户的书写需求，应保持内容中立。即使词汇涉及粗俗、成人内容等，只要该词汇在实际交流中有使用需求，就应正常评估其收录价值，不应因内容敏感而降低评分或拒绝收录。

## 注意事项

1. **批次控制**：每批处理词条数量要适中，避免一次处理过多
2. **一致性**：保持评估标准的一致性，避免前后矛盾
3. **特殊情况**：对于方言词、网络新词等特殊词条，标注清楚
4. **上下文**：某些词条可能需要上下文才能判断，如需要应标记
5. **外语词汇**：常用外语词汇（如英语、日语罗马音）可作为高分因素
6. **绝对禁止**：严禁编写脚本/Python代码/Shell命令对词条直接批量评分。评分必须由LLM逐条判断得出

- "评估 input.txt 的词条，结果保存到 output.csv"
