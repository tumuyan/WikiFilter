# 输入法词条评估技能

## 概述

此技能用于评估文本文件中的词条是否适合列入输入法词典。通过分批次处理，对每个词条进行多维度的适用性评估。

**适用范围：** 以中国大陆地区简体中文字形为主，评估词条是否符合简体中文规范和使用习惯。

## 使用场景

当用户需要：
- 批量评估词条是否适合输入法词典
- 筛选高质量的输入法候选词
- 检查词条的常用度、规范性和适用性
- 处理大量词条数据时需要LLM辅助判断

## 工作流程

### 1. 获取处理参数

首先确认以下信息：
- **输入文件**：要处理的文本文件路径
- **输出文件**：评估结果保存的CSV文件路径（如未指定则默认输出到对话中）
- **批次大小**：每批处理的词条数量（建议20-100条）
- **起始位置**：从第几行开始处理（默认第1行）
- **特殊要求**：是否有特定评估标准

### 2. 读取词条内容

使用 `read_file` 工具读取指定范围的词条内容。

### 3. 执行评估

对每个词条按以下维度进行评估：

#### 评估维度

| 维度 | 说明 | 权重 |
|------|------|------|
| **常用性** | 该词条在日常使用中的频率 | 高 |
| **规范性** | 是否为标准书写形式，无明显错别字 | 高 |
| **完整性** | 是否为完整词汇，非残缺片段 | 中 |
| **歧义性** | 是否容易与其他词混淆 | 中 |
| **专业性** | 是否为过于专业的术语 | 低 |

#### 评估标准

**地区与字形规范：**
- 以中国大陆简体中文为标准
- 繁体字词条需标注，建议转换为简体
- 异体字、旧字形按简体规范处理
- 港台用语需标注地区特性

**适合收录的词条特征：**
- 常用词汇、短语、成语（简体中文规范写法）
- 地名、人名、机构名等专有名词
- 网络流行语（广泛使用的）
- 专业领域常用术语

**不适合收录的词条特征：**
- 包含错别字或非标准写法
- 过于生僻的词汇
- 残缺不完整的片段
- 包含特殊字符或符号
- 重复或冗余的词条
- 过长或过于复杂的句子

### 4. 词条分类

评估时需对词条进行分类，详见 `references/evaluation-criteria.md`：

- **基础分类**（必填，单选）：名词/动词/形容词/副词/代词/数词/量词/拟声词/叹词/成语/专有名词/短语/网络用语/其他
- **其他标签**（可选，多选）：#口语 #书面语 #方言 #专业术语 #新词 #生僻 #多音字 #易错 #繁体 #港台用语 #和字
### 5. 输出格式

评估结果以CSV格式输出：

```csv
置信度,评分,词条,基础分类,其他标签,原因
9,95,手机,名词,#口语,极常用词汇，简体中文规范
8,88,运行,动词,,常用动词，使用频率高
7,55,畫面,名词,#繁体,繁体字词条，建议转换为"画面"
```

**CSV字段说明：**
- 置信度：评估置信度（0-10分）
- 评分：适用性评分（0-100分）
- 词条：原始词条内容
- 基础分类：必填分类
- 其他标签：可选标签，多个标签用空格分隔
- 原因：简要说明

#### 评分说明

**适用性评分（0-100分）：**
- 90-100分：强烈推荐收录，适合度极高
- 70-89分：推荐收录，适合度较高
- 50-69分：可考虑收录，适合度一般
- 30-49分：不建议收录，适合度较低
- 0-29分：不推荐收录，不适合

**置信度评分（0-10分）：**
- 8-10分：判断依据充分，结果可靠
- 5-7分：判断依据较充分，结果较可靠
- 0-4分：判断依据不足，建议人工复核

### 6. 汇总统计

每批次结束后输出统计：
- 高分词条（≥70分）数量
- 中分词条（50-69分）数量
- 低分词条（<50分）数量
- 平均评分
- 平均置信度
- 建议操作

### 7. 输出目标

评估结果的处理方式：
- **指定输出文件**：将CSV结果追加写入指定文件
- **未指定输出文件**：在对话中直接展示CSV内容，用户可复制保存

## 评估Prompt模板

```
请评估以下词条是否适合列入简体中文输入法词典：

词条：{词条内容}

评估要求：
1. 地区规范：以中国大陆简体中文为标准，繁体字需标注#繁体
2. 基础分类：选择最匹配的分类（名词/动词/形容词/成语/专有名词/短语/网络用语/其他）
3. 其他标签：可选补充标签（#口语 #书面语 #方言 #专业术语 #新词 #生僻 #多音字 #易错 #繁体 #港台用语 #和字）
4. 适用性评分：0-100分（考虑常用性、规范性、完整性、歧义性、专业性）
5. 置信度评分：0-10分（判断依据的充分程度）
6. 简要原因（一句话说明评分依据）
```

## 注意事项

1. **批次控制**：每批处理词条数量要适中，避免一次处理过多
2. **一致性**：保持评估标准的一致性，避免前后矛盾
3. **特殊情况**：对于方言词、网络新词等特殊词条，标注清楚
4. **上下文**：某些词条可能需要上下文才能判断，如需要应标记

## 使用示例

用户请求示例：
- "帮我评估 dict.txt 文件中的词条，每批20条"
- "从第100行开始评估，检查是否适合输入法词典"
- "评估 input.txt 的词条，结果保存到 output.csv"
- "这批词条哪些适合加入输入法词库？"

执行流程：
1. 确认输入文件、输出文件（可选）、批次大小、起始位置
2. 读取指定范围的词条
3. 逐一评估每个词条
4. 以CSV格式输出评估结果
5. 提供汇总统计和建议
